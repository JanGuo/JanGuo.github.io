Linux安装Docker与使用

常用命令

  • 查看linux的ip地址
    ip addr

  • 1、检查内核版本，必须是3.10及以上

    uname -r

  • 2、安装docker

    yum install docker

  • 3、输入y确认安装

  • 4、启动docker

    systemctl start docker

    docker -v

  • 5、开机启动docker

    systemctl enable docker

  • 6、停止docker

    systemctl stop docker

1、搜索镜像

docker search tomcat

2、拉取镜像

docker pull tomcat

3、根据镜像启动容器

docker run –name mytomcat -d tomcat:latest

4、查看运行中的容器

docker ps

5、 停止运行中的容器

docker stop 容器的id

6、查看所有的容器

docker ps -a

7、启动容器

docker start 容器id

8、删除一个容器

docker rm 容器id

9、启动一个做了端口映射的tomcat

docker run -d -p 8888:8080 tomcat

-d：后台运行

-p: 将主机的端口映射到容器的一个端口 主机端口:容器内部的端口

10、为了演示简单关闭了linux的防火墙

service firewalld status ；查看防火墙状态

service firewalld stop：关闭防火墙

11、查看容器的日志

docker logs container-name/container-id

更多命令参看

https://docs.docker.com/engine/reference/commandline/docker/
可以参考每一个镜像的文档

【Docker加速】- 解决docker pull镜像速度慢的问题

参考文献https://blog.csdn.net/shenzhen_zsw/article/details/74277518

只需要按照操作一遍即可,非常方便

安装redis

docker pull redis

docker images

docker run -d -p 6379:6379 –name myredis {id}

安装rabbitmq

docker pull rabbitmq:3-management

docker run -d -p 5672:5672 -p 15672:15672 –name myrabbitmq {id}

  • 可以通过ip:15672 来访问主页面 默认账号guest 密码guest

安装elasticsearch

直接使用

docker pull elasticsearch

会出现 错误Error response from daemon: manifest for elasticsearch:latest not found

所以使用

docker pull elasticsearch:5.1.1

由于内存不够使用所以添加-e 来限制空间

docker run -e ES_JAVA_OPTS=”-Xms256m -Xmx256m” -d -p 9200:9200 -p 9300:9300 –name MyES {id}

执行命令后elasticsearch自动关闭

通过命令 docker logs {id}发现原因 

max virtual memory areas vm.max_map_count [65530] is too low, increase to at least [262144]

(假的)解决办法：

1、切换到root用户修改配置sysctl.conf

vi /etc/sysctl.conf

添加下面配置：

** vm.max_map_count=655360**

并执行命令：

sysctl -p

然后，重新启动elasticsearch，即可启动成功。

真正的解决方法

参考文献https://blog.csdn.net/dsx1134500455/article/details/89531146

使用之后发现还是没有什么作用
之后经过了一个多小时的百度终于找到了一个解决的办法就是pull一个5.6.15版本的elasticsearch

在使用run命令就ok了
